Catálogo

Mostruário Digital — Loja Ambiente

O desafio

A Loja Ambiente trabalhava com um mostruário físico de tecidos, sintéticos e impermeáveis — o que limitava o atendimento a quem estava na loja e dificultava a consulta rápida por parte da equipe comercial e dos clientes à distância.

A solução

Um mostruário digital completo, rodando em um único arquivo HTML — sem servidor, sem banco de dados, sem custo de hospedagem. O cliente abre o link (ou o arquivo) e navega pelo catálogo inteiro: mais de 170 tecidos organizados em três categorias, com busca por código, cor ou linha, fotos em alta definição e detalhes técnicos de cada material.

Para a loja, o sistema também funciona como uma ferramenta de gestão de catálogo: um modo de edição protegido por senha permite adicionar, duplicar, editar e remover tecidos diretamente pela interface, sem precisar de programador para cada atualização — bastando salvar o arquivo (localmente ou publicando direto no GitHub) para a nova versão ir ao ar.

O resultado

Um catálogo sempre atualizado, acessível de qualquer lugar, com uma experiência de navegação pensada para o cliente final (fotos grandes, marca d'água discreta, botão de favoritos e envio direto pelo WhatsApp) e um fluxo de manutenção simples para a equipe da loja.

Ficha técnica

Tipo: aplicação web single-file (HTML + CSS + JavaScript puro)
Stack: sem frameworks, sem dependências externas, sem backend — 100% client-side
Armazenamento: dados do catálogo embutidos no próprio arquivo HTML, como um banco de dados portátil

Funcionalidades — visão do cliente

  • Catálogo organizado por categorias (Sintéticos, Tecidos, Impermeáveis)
  • Busca em tempo real por código, cor, linha ou referência do fabricante
  • Ficha de detalhe: foto ampliada, código interno, referência do fabricante, material e selos (ex: repelente à água, pet friendly)
  • Sistema de favoritos, com lista dedicada e envio direto da seleção por WhatsApp
  • Marca d'água discreta da marca sobre as fotos
  • Carregamento otimizado de imagens (lazy loading)

Funcionalidades — modo administrador

  • Acesso protegido por senha (PIN configurável)
  • Cadastro de novos tecidos com upload de foto
  • Edição completa de qualquer item (código, cor, linha, material, categoria, selos)
  • Duplicação de item (base para variações de cor/linha)
  • Exclusão de item
  • Publicação de nova versão: salvar localmente (File System Access API) ou publicar direto num repositório GitHub via API
  • Separação entre arquivo de trabalho (com senha real, editável) e arquivo exportado para o cliente (interface de edição bloqueada)

Detalhes técnicos de implementação

  • Compressão e redimensionamento automático de imagens para manter o arquivo dentro do limite de upload do GitHub (25 MB)
  • Interface responsiva com modal de detalhe rolável, adaptado para catálogos extensos
  • Arquitetura pensada para resiliência: proteções contra estados inconsistentes ao salvar (garante que o arquivo exportado sempre abra na tela inicial, nunca "travado" em modo edição)
Sistema

Notas de Recibos — Importação

Funcionalidades

  • Formulário de nota de compra/recibo com layout profissional para impressão em A4
  • Sistema de login com PIN de acesso
  • Tabela de itens dinâmica: adicionar/remover linhas de produtos
  • Cálculo automático: quantidade × valor unitário = valor total por item, com soma geral em tempo real
  • Campos de desconto (em R$ ou %) e entrada (valor já pago), ambos opcionais e com exibição condicional na impressão (só aparecem no PDF se preenchidos)
  • Máscaras automáticas de CPF/CNPJ e telefone
  • Numeração automática incremental de notas (via localStorage)
  • Preenchimento automático da data atual
  • Geração de PDF via impressão nativa do navegador (compatível com celular)
  • Função "Limpar/Nova venda" que reseta o formulário mantendo configurações fixas (WhatsApp da loja)
  • Marca d'água e identidade visual da empresa integradas ao layout

Diferenciais técnicos

  • Responsivo e otimizado para impressão (regras @media print dedicadas)
  • Tratamento de erros defensivo (ex: fallback caso localStorage esteja bloqueado, comum em navegadores como o do WhatsApp)
  • Parsing de valores monetários brasileiros (aceita "1.200,50", "1200.50" etc.)
  • UX pensada para uso no dia a dia de uma loja física, sem precisar de backend ou instalação
